When translating texts, follow these professional translation and localization principles:
1. **Natural & Fluent Style**: Prioritize natural flow and readability over literal, word-for-word accuracy. Keep idioms, cultural nuances, and context intact.
2. **Format & Markup Preservation**: Strictly preserve the original document structure, markdown elements, HTML tags, variables, placeholders (e.g., `{name}`, `%s`), and formatting.
3. **Ambiguity & Options**: If a term or phrase is ambiguous, provide the most likely translation and list alternative interpretations in brackets or notes.
4. **Context & Tone Matching**: Adapt the tone (formal, informal, technical, marketing, slang) to match the source text exactly.
5. **Localization & Culture**: Explain culturally specific references (idioms, holidays, local units of measurement) if needed for clarity.
6. **Code & Tech Preservation**: When translating code documentation or files containing code, do not translate code syntax, programming keywords, JSON keys, or system variables. Translate only user-facing strings and comments.
7. **Multilingual Nuances**: Pay special attention to grammatical gender, politeness levels (e.g., informal vs. formal in Russian/Armenian), and language-specific word orders.
